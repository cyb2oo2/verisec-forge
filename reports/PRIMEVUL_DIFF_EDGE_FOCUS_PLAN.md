# PrimeVul Diff Edge-Focus Experiment Plan

This plan turns the paired diff failure analysis into a concrete next experiment.

## Motivation

The current paired diff-only detector is the strongest PrimeVul mainline, but its remaining errors are not evenly distributed across diff sizes.

- Overall deduplicated paired eval: `accuracy = 0.8158`, `recall = 0.8022`, `specificity = 0.8294`
- Error mix: `153` false positives and `177` false negatives
- `00-02` changed-line bucket error rate: `0.2224`
- `26+` changed-line bucket error rate: `0.2830`

This suggests that the next useful training question is not "more data everywhere", but whether targeted exposure to very small and very large diffs improves the hardest regions without destroying the global paired score.

## Dataset Construction

Generated by:

```bash
python scripts/build_primevul_diff_bucket_slices.py
```

Artifacts:

- train focus set: `data/processed/secure_code_primevul_pair_diff_edge_focus_train_balanced_3000_metadata.jsonl`
- eval bucket slices: `data/processed/primevul_diff_bucket_slices/*.jsonl`
- summary: `reports/secure_code_primevul_pair_diff_bucket_slices_summary.json`

The focused train set has `3000` examples. It keeps label balance and targets `60%` of training traffic at the high-error edge buckets:

- edge buckets: `00-02`, `26+`
- background buckets: `03-05`, `06-10`, `11-25`
- edge oversampling uses replacement because the original train set has only `511` vulnerable and `528` safe edge examples.
- the resulting focused set has `3000` rows, `2239` unique ids, and `761` duplicate rows introduced by replacement oversampling.
- it covers `1809` unique pair keys.

## Success Criteria

Treat the experiment as a win only if it improves the target buckets without breaking the mainline.

- `00-02` bucket accuracy should improve over the current failure-analysis baseline.
- `26+` bucket accuracy should improve over the current failure-analysis baseline.
- Full deduplicated eval balanced accuracy should stay close to the current `0.8158` baseline, ideally within `-0.02` or better.
- False positives and false negatives should remain balanced; a one-sided recall/specificity collapse is not a real improvement.

## Next Step

Run the edge-focus training config, then evaluate the full deduplicated eval plus the two edge buckets. If this helps, the follow-up is a smaller sweep over `edge_share` values such as `0.4`, `0.6`, and `0.8`.

## Separate Metadata Ablation

A second reviewer-facing control is now prepared: `diff_no_metadata`. This removes `Project`, `CVE`, and `CWE` lines from the prompt and leaves only the task instruction plus unified diff.

- train: `configs/cls_secure_code_primevul_qwen15bcoder_lora_pair_diff_no_metadata_3000_v1.json`
- eval: `configs/cls_eval_secure_code_primevul_qwen15bcoder_lora_pair_diff_no_metadata_3000_v1_eval1800.json`
- train summary: `reports/secure_code_primevul_pair_diff_no_metadata_train_balanced_3000_summary.json`
- eval summary: `reports/secure_code_primevul_pair_diff_no_metadata_eval_balanced_1800_summary.json`

This is the most important next control. If `diff_no_metadata` remains near the `0.80+` range, the paired diff result is much harder to dismiss as metadata leakage.
